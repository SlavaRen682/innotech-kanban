# Пользовательские сценарии

## Регистрация и вход

Пользователь создает аккаунт по имени, логину и паролю. После регистрации система сразу создает личный workspace и проект `Канбан`, чтобы можно было начать работу без ручной настройки.

## Команда в workspace

Владелец workspace добавляет участника по логину. Такой формат выбран для учебного проекта: он понятен на защите, не требует почтовой инфраструктуры и показывает реальную механику совместной доски.

## Создание задачи

В карточке задаются:

- название и описание;
- проект;
- исполнитель;
- приоритет;
- срок;
- тег;
- чеклист готовности;
- ссылки на материалы.

Чеклист работает как критерий готовности. Если пункты не выполнены, задачу нельзя переместить в `Готово`.

## Работа с задачей

Правая панель показывает выбранную задачу, ее атрибуты, чеклист, материалы, комментарии и историю. Комментарии нужны для решений и уточнений, история - для восстановления контекста: кто создал, переместил, отложил или вернул задачу.

## Отложенные задачи

Если задача заблокирована внешней причиной, ее можно отложить с объяснением. Она исчезнет из активного потока, но останется в режиме `Отложено`. Оттуда ее можно вернуть на доску, когда причина исчезла.

## Поиск и фильтры

Верхняя панель позволяет искать задачи по тексту и фильтровать по тегу. Переключатель `Активные / Отложено` разделяет текущую работу и заблокированные задачи.
