# Активный контекст

## Текущая задача

Сделать канбан-доску на русском языке с исследованной уникальной фишкой, без моковых карточек и без демо-режима.

## Ветка

`codex/kanban-aging-demo`

## Состояние реализации

Статические файлы приложения:

- `index.html`
- `src/kanban-core.js`
- `src/app.js`
- `src/styles.css`
- `tests/kanban-core.test.mjs`

Уникальная фишка: Ассистент фокуса ранжирует реальные незакрытые карточки по зависанию, блокировке, WIP-перегрузу, приоритету, риску срока и размеру.

Текущий UI:

- commandbar с метриками и кнопкой `Новая задача`;
- компактный блок `Что закрыть первым`;
- selected-card strip с действиями;
- полноширинная канбан-доска из 5 колонок;
- пустые состояния для первого запуска и пустых колонок.
- форма создания карточки использует кастомные segment controls для `Приоритет` и `Размер`, без нативных dropdown.

Приложение стартует пустым. Карточки появляются только после ручного создания и сохраняются в `localStorage` под ключом `kanban-flow-board-state-v1`. Фиксированной даты, `+1 день`, сброса сценария, seed-данных и внешнего font import больше нет.

Backend, авторизация, база данных и multi-user sync не добавлены. Для них нужно отдельное архитектурное решение.

## Проверено

- `npm run build`: lint + 7 Node tests проходят.
- Поиск по старым demo/mock строкам пустой.
- Playwright MCP:
  - чистый первый запуск дает 0 карточек и 5 колонок;
  - добавление карточки через форму работает;
  - действие `Вперед` переводит созданную карточку в `В работе`;
  - storage key `kanban-flow-board-state-v1` содержит созданную карточку;
  - desktop 1440px и mobile 390px без horizontal overflow;
  - console чистая после удаления внешнего шрифта.
- После замены dropdown Playwright подтвердил `selectCount=0`, сохранение `priority=high` и `size=L`, desktop/mobile форму без overflow и чистую консоль.
