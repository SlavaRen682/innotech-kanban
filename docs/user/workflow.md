# Пользовательские сценарии

## Регистрация

Пользователь регистрируется по имени, логину и паролю. После регистрации система создает `Мой workspace` и проект `Канбан`.

## Добавление участника

Владелец добавляет пользователя в workspace по логину. Почта не используется: для учебного проекта это проще показать и легче обосновать.

## Создание задачи

В задаче можно указать:

- название и описание;
- проект;
- исполнителя;
- приоритет;
- срок;
- тег;
- чеклист;
- ссылки;
- файлы с компьютера.

Файл загружается на backend, появляется в материалах задачи и открывается по ссылке.

## Контроль готовности

Если в чеклисте есть невыполненные пункты, задачу нельзя закрыть в `Готово`. Это защищает команду от формального закрытия сырой работы.

## Комментарии и история

Комментарии сохраняют обсуждение, а история показывает действия: создание, изменение, выполнение чеклиста, комментарии, отложение и возврат.

## Отложенные задачи

Задачу можно отложить с причиной и датой возврата. Она исчезает из активной доски и отображается в режиме `Отложено`, откуда ее можно вернуть.
